# Как я построил систему мониторинга температуры за 1600 рублей (и почему это лучше готовых решений)

## Предисловие

В каждом доме есть холодильник. А в каждом холодильнике — риск выйти из строя. Обычно мы узнаём об этом, когда из морозилки начинает течь вода, а продукты безнадёжно испорчены. 
Я решил, что с меня хватит сюрпризов, и построил систему, которая сама сообщит мне о проблеме — ещё до того, как я открою дверцу.

**Бюджет:** ~1600 рублей  
**Результат:** Полностью автономный мониторинг с графиками, уведомлениями в Telegram и Email, историей данных и JSON для интеграции.

### Схема подключения (очень простая)

Датчики подключаются **параллельно** к одному пину GPIO23:
*Важно:** Резистор 4.7 кОм — не опция. Без него датчики не будут работать из-за особенностей протокола 1-Wire.

### Почему DS18B20?

- Цифровой протокол 1-Wire — десятки датчиков на одном пине
- У каждого уникальный 64-битный адрес (не нужно думать о коллизиях)
- Диапазон от -55°C до +125°C — с запасом покрывает морозилку
- Точность ±0.5°C — более чем достаточно для бытового мониторинга

[Даташит DS18B20](https://www.alldatasheet.com/datasheet-pdf/view/433921/MAXIM/DS18B20.html)

### Три слоя системы

#### 1. ESP32 — «периферия»
- Измеряет температуру каждые 10 секунд
- Отдаёт JSON по адресу `/data`
- Имеет веб-интерфейс с авторизацией
- Доступен по имени `esp32-temp.local` (mDNS)

**Пример ответа ESP32:**
json
{"sensor1":23.5,"sensor2":24.1}

🔥 ALERT: High Temperature Alert - Fridge
Температура: +9.2°C
Допустимый диапазон: +2°C до +8°C
Время: 2026-06-07 15:30:00

Надёжность
ESP32 переживает отключение Wi-Fi — автоматически переподключается

Python-мост в автозагрузке Windows
InfluxDB + Grafana запускаются как службы
Данные не теряются — контейнер Docker сохраняет всё на диск

⚠️ Три ошибки, которые я совершил (и вы не совершите)
1. Забыл резистор 4.7 кОм
Датчики показывали -127°C три часа, пока я не перечитал даташит. Резистор стоит копейки, а без него 1-Wire не работает.

2. Использовал localhost в ESP32
В коде написал INFLUXDB_URL = "http://localhost:8086". ESP32 воспринимает localhost как себя самого, а не как сервер в сети. Нужен реальный IP.

3. Пытался отправить два датчика в одной точке InfluxDB
Структура Point не умеет хранить два поля в одной временной метке — пришлось создавать отдельный Point для каждого датчика.

## 🔧 Компоненты, которые нужно настроить на автозапуск

1. **Docker Desktop** (с контейнером InfluxDB)
2. **Grafana**
3. **Python-мост** (ESP32 → InfluxDB)
---

## 1️⃣ Docker Desktop — автозапуск

### Встроенная настройка Docker:
1. Открыть **Docker Desktop**
2. Нажать на иконку шестерёнки (⚙️) **Settings**
3. Перейти в раздел **General**
4. Убедится, что опция **"Start Docker Desktop when you log in"** включена ✅
5. Автоматический запуск контейнера InfluxDB:

    docker save -o influxdb-2.7.tar influxdb:2.7
    docker load -i influxdb-2.7.tar
    docker run -d --name influxdb -p 8086:8086 influxdb:2.7

### Дополнительно: автозапуск контейнера InfluxDB
Контейнер `influxdb2` должен автоматически запускаться вместе с Docker. Добавьте флаг `--restart` при создании (если контейнер уже есть — обновите):

   docker update --restart unless-stopped influxdb2
Или удалите и создайте заново:
   docker rm influxdb2
   docker run --detach --name influxdb2 --restart unless-stopped -p 8086:8086 influxdb:2.7

## 2️⃣ Grafana — автозапуск

### На Windows (служба):
Grafana обычно устанавливается как Windows-служба и должна запускаться автоматически.

**Проверьте:**
1. Откройте `services.msc` (`Win + R` → `services.msc`)
2. Найдите службу **Grafana** (или `grafana-server`)
3. Убедитесь, что **"Тип запуска"** установлен как **"Автоматически"** ✅
Если нет — нажмите правой кнопкой → **Свойства** → Тип запуска: **Автоматически** → Применить.
---

## 3️⃣ Python-мост — автозапуск

Это самый важный компонент. У вас есть скрипт `esp32_influxdb_bridge.py`, который нужно запускать каждые 10 секунд. Есть несколько способов:
### ⭐ Способ А: Планировщик задач Windows (рекомендуется)
Создайте задачу, которая запускает скрипт при входе в систему.

**Инструкция:**
1. Нажмите `Win + R`, введите `taskschd.msc` и нажмите Enter
2. В правой панели нажмите **"Создать задачу..."**
3. На вкладке **"Общие"**:
   - **Имя**: `ESP32 Temperature Bridge`
   - **Включить** ✅ "Выполнять с наивысшими привилегиями"
   - **Настроить для**: `Windows 10`

4. На вкладке **"Триггеры"**:
   - Нажмите **"Создать..."**
   - **Начать задачу**: `При входе в систему`
   - **Пользователь**: `Любой пользователь` (или выберите конкретного)
   - Нажмите **ОК**

5. На вкладке **"Действия"**:
   - Нажмите **"Создать..."**
   - **Действие**: `Запустить программу`
   - **Программа или сценарий**: `D:\anaconda\pythonw.exe` (путь к вашему Python)
   - **Добавить аргументы**: `C:\dev\temp_monitor\esp32_influxdb_bridge.py`
   - **Рабочая папка**: `C:\dev\temp_monitor`
   - Нажмите **ОК**

6. На вкладке **"Условия"** (опционально):
   - Снимите галочку ❌ "Запускать только при питании от электросети" (если хотите работу на батарее)

7. На вкладке **"Параметры"**:
   - Включите ✅ "При сбое выполнения перезапускать через" → `1 минута`
   - Количество попыток: `3`
   - Включите ✅ "Если задача не выполняется, принудительно остановить её через" → `30 минут`

8. Нажмите **ОК** и введите пароль вашей учётной записи (если потребуется)






### Способ Б: Bat-файл + Автозагрузка
Создайте `.bat` файл в папке автозагрузки.

1. Создайте файл `start_esp32_bridge.bat` с содержимым:
```batch
@echo off
cd /d C:\dev\temp_monitor
D:\anaconda\python.exe esp32_influxdb_bridge.py
```

2. Нажмите `Win + R`, введите `shell:startup` и нажмите Enter
3. Скопируйте созданный `start_esp32_bridge.bat` в эту папку

**Минус:** При каждом запуске будет открываться чёрное окно консоли.

### Способ В: Служба Windows (через NSSM)

[**NSSM**](https://nssm.cc/) (Non-Sucking Service Manager) позволяет превратить Python-скрипт в настоящую Windows-службу.

```bash
nssm install ESP32Bridge D:\anaconda\python.exe C:\dev\temp_monitor\esp32_influxdb_bridge.py
nssm start ESP32Bridge
```

## 4️⃣ ESP32 — автоподключение

Ваша ESP32 настроена на автоматическое подключение к Wi-Fi при включении питания. Она не требует дополнительных настроек.
**Рекомендация:** Подавайте питание на ESP32 через USB-порт компьютера или от стабильного блока питания, который включён в розетку с защитой от скачков.

## 📊 Финальная проверка

После того как вы настроите всё выше, **перезагрузите компьютер** и проверьте:

1. **Docker Desktop** запустился автоматически ✅
2. **Grafana** доступна по адресу `http://localhost:3000` ✅
3. **InfluxDB** работает (проверьте через `docker ps`) ✅
4. **Python-мост** запустился (по сообщениям в консоли или задаче в планировщике) ✅
5. **ESP32** горит синим светодиодом (или другим индикатором) и доступна в сети ✅
6. **Grafana дашборд** обновляется свежими данными ✅
7. **Telegram + Email** оповещения приходят при аварийной температуре ✅

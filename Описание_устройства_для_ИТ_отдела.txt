Техническое описание устройства для предоставления в ИТ-отдел :

1. Устройство с MAC-адресом `ТВОЙ_МАК_АДРЕС` и статическим\динамическим IP-адресом представляет собой специализированный измерительный контроллер на базе платформы ESP32, предназначенный исключительно для локального мониторинга температуры в двух холодильных камерах.  
2. На устройстве не установлено и не может быть установлено никакое программное обеспечение для передачи данных в сеть интернет – вся работа ограничена локальной офисной сетью.  
3. Контроллер не имеет SIM-карты, модуля 4G/5G, Bluetooth-стека в активном режиме, а также не поддерживает встроенных облачных сервисов (например, Tuya или Blynk).  
4. Единственный исходящий сетевой трафик – это HTTP GET-запросы по протоколу IPv4, которые отправляются строго на статический IP-адрес (офисный компьютер сотрудника "ТВОЯ_КОМПАНИЯ") порт `8086`.  
5. Формат передаваемых данных – короткие JSON-пакеты вида `{"sensor1":23.5,"sensor2":24.1}`, содержащие только текущие показания температуры; никакая иная информация (серийные номера ПК, логины, данные документов) не собирается и не передаётся.  
6. Устройство не инициирует соединений на порты, отличные от `80` (веб-сервер для локального просмотра) и `8086` (отправка в InfluxDB на тот же компьютер).  
7. Веб-интерфейс контроллера доступен только по протоколу HTTP без SSL и защищён паролем (логин/пароль `admin/esp32`), что исключает несанкционированный доступ изнутри сети.  
8. В контроллере отсутствуют механизмы автообновления прошивки, DNS-запросы к произвольным доменам или возможность выполнения скриптов – прошивка статична и загружена один раз через USB.  
9. Все собранные данные о температуре остаются внутри офисной сети: они записываются в локальную базу InfluxDB на том же компьютере офисном компьютере сотрудника "ТВОЯ_КОМПАНИЯ" и визуализируются только через локальный сервер Grafana (порт `3000`).  
10. Таким образом, устройство функционирует как обычный WiFi-датчик, не создаёт внешних соединений, не передаёт данные за пределы предприятия и может быть безопасно подключено к производственной Wi-Fi сети с ограничением исходящего трафика на файрволе.

При необходимости ИТ-отдел может добавить правило блокировки всего исходящего трафика для устройства, оставив разрешённым только доступ к `IP-адрес сотрудника ТВОЯ_КОМПАНИЯ:8086` – устройство продолжит работать штатно.
